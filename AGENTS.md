# AGENTS.md

本文件定义 DynamicD 仓库内所有 AI Agent 的统一开发规范。适用于：规划、实现、测试、评审、发布支持类 Agent。

## 1. 角色与职责
- `Planner`：基于规范输出可执行任务拆分与验收标准，不直接提交功能代码。
- `Implementer`：按任务实现代码与文档，保证可编译、可测试、可回滚。
- `Reviewer`：优先发现缺陷、回归风险、需求遗漏与测试缺口。
- `Verifier`：执行测试、复现实验、验证需求追踪。
- `Release`：整理变更说明、兼容性说明、发布检查项。

## 2. 统一行为规则
- 所有 Agent `MUST` 先读取：
  - `docs/spec/dynamicd-spec.md`
  - `docs/requirements/dynamicd-ears.md`
  - `docs/design/implementation-plan.md`
  - `DD.txt`
  - `yuz.txt`
- 所有变更 `MUST` 显式关联至少一个需求 ID（如 `DD-AGT-003`）。
- 未经授权，`MUST NOT` 直接执行危险操作（删除关键模块、权限写入、危险命令）。
- 若需求与实现冲突，`MUST` 先提交差异说明并等待策略确认，不得静默偏离规范。

## 3. 开发流程协议
1. 澄清目标：确认变更目标、影响边界、对应 EARS ID。
2. 设计变更：给出最小可行实现和失败回退方案。
3. 实施变更：优先局部改动，避免无关重构。
4. 自检验证：至少执行相关单元/集成测试或给出无法执行原因。
5. 交付报告：输出修改文件、行为变化、风险点、回滚步骤。

## 4. 代码与文档质量门槛
- `MUST` 保持接口清晰、错误可诊断、日志可追踪。
- `MUST` 维护结构化错误信息（错误码、位置、建议）。
- `SHOULD` 在复杂逻辑附近添加简短解释性注释。
- `MUST NOT` 引入与任务无关的大范围格式化或重命名噪音。
- 文档改动 `MUST` 同步更新需求追踪关系。

## 5. Agent 专项约束（DynamicD）
- Agent 相关改动 `MUST` 满足工具链约束：`read/search/patch/compile/load/rollback` 可审计。
- 补丁策略 `MUST` 遵循：AST Patch > 结构化 Token Patch > 文本兜底。
- 危险动作 `MUST` 具备二次确认路径与拒绝路径。
- 每次自动改动 `MUST` 生成回滚点标识（或可恢复策略）。

## 6. 权限与安全
- `MUST` 使用最小权限原则；缺失权限时拒绝执行并说明缺失节点。
- `MUST` 记录高风险动作审计字段：
  - `requestId`
  - `operator`
  - `action`
  - `target`
  - `decision`
  - `timestamp`
  - `snapshotId`（如适用）
- `MUST NOT` 绕过沙箱、审计或权限检查路径。

## 7. 测试与验收
- 每个实现任务 `MUST` 提供验收映射：`DD-* -> AT-*`。
- `P0` 变更 `MUST` 包含回归验证。
- 若测试无法执行，`MUST` 明确列出原因、风险与后续补测建议。

## 8. 协作与冲突处理
- 多 Agent 并行时，`MUST` 先声明各自文件所有权范围。
- 发现他人未合并改动时，`MUST` 避免覆盖，必要时改为增量补丁。
- 出现冲突时，`MUST` 优先保留已验证行为并回到需求 ID 对齐。

## 9. 提交与报告模板
### 9.1 提交信息
`<type>(<scope>): <summary> [DD-XXX-YYY]`

示例：
`feat(agent): add compile-load guarded pipeline [DD-AGT-003]`

### 9.2 交付报告
- 需求：`DD-...`
- 变更：文件与行为摘要
- 验证：测试项与结果
- 风险：潜在影响
- 回滚：步骤或快照 ID

## 10. 禁止事项
- 禁止在未说明的情况下修改需求语义。
- 禁止绕过权限直接执行高危命令。
- 禁止提交无法解释来源的自动生成大改动。
- 禁止删除审计与安全控制逻辑（除非显式需求）。
